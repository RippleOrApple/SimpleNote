# SimpleNote 产品需求文档

> **历史文档。**
>
> 本文描述 v1/MVP 阶段的产品范围，保留用于回溯项目演进。当前 V2 开发请以 `../../superpowers/specs/2026-07-17-ticktick-style-v2-design.md` 和 `../../superpowers/plans/2026-07-19-phase-3-4-habits-statistics-sync-plan-DO-NOT-DELETE.md` 为准。文中关于旧待办和 V1 局域网同步的内容不代表当前生产状态。

## 1. 项目定位

SimpleNote 是一款面向个人用户的 Windows / Android 双端本地笔记与待办应用。它强调离线可用、界面简洁、主题高度自定义，并支持同一局域网内的设备数据同步。

项目目标不是做一个功能庞大的云笔记平台，而是做一个小而完整、体验清爽、可以独立部署和演示的本地效率工具。

## 2. 核心价值

- 本地优先：数据默认保存在用户自己的电脑或手机上，离线也能完整使用。
- 双端可用：Windows 与 Android 使用同一套核心能力，适合日常跨设备记录。
- 简洁高效：笔记、待办、搜索、编辑、预览等操作路径尽量短。
- 高度自定义：用户可以自由设置背景色、主按钮色、文字色，并保存主题方案。
- Markdown 友好：支持 Markdown 编辑与预览，适合学习、开发、复盘、计划等场景。
- 局域网同步：电脑和手机连接同一个网络时，可以互相同步数据。

## 3. 目标用户

### 3.1 主要用户

- 在校学生：记录课程笔记、实验过程、论文阅读、待办任务。
- 开发者：记录技术笔记、代码片段、项目计划、开发任务。
- 个人效率工具用户：希望拥有轻量、离线、无账号压力的笔记待办工具。

### 3.2 用户痛点

- 云笔记产品功能复杂，打开和记录成本偏高。
- 待办和笔记分散在不同应用里，不方便统一管理。
- 手机和电脑之间传输文本、计划、草稿不够顺手。
- 很多应用主题自定义能力弱，不符合个人审美或使用习惯。
- 有些笔记工具离线体验不好，或强依赖账号和云服务。

## 4. 使用场景

### 4.1 学习记录

用户在电脑端编写课程笔记，使用 Markdown 记录标题、列表、链接和代码块。晚上用手机查看同一份笔记，并补充临时想法。

### 4.2 项目开发

用户在笔记中记录项目需求、技术方案、接口说明，并在待办中拆解开发任务，例如“完成 Markdown 预览”“实现局域网同步”“打包 Android 安装包”。

### 4.3 日常计划

用户在手机上快速创建待办，回到电脑后同步数据，在大屏上整理当天计划和长期事项。

### 4.4 个性化工作台

用户根据自己的喜好设置背景色、按钮色、文字色，保存为主题方案，在不同环境下切换浅色、深色或护眼主题。

## 5. 功能范围

### 5.1 笔记功能

#### 必须支持

- 新建笔记。
- 编辑笔记标题和正文。
- 删除笔记。
- 搜索笔记标题和正文。
- 笔记分类或标签。
- 按更新时间展示笔记列表。
- 本地保存笔记内容。

#### 建议支持

- 置顶笔记。
- 归档笔记。
- 最近编辑记录。
- 空状态提示。

#### 暂不支持

- 多人协作编辑。
- 云端分享链接。
- 复杂富文本编辑器。
- 附件管理。

### 5.2 待办功能

#### 必须支持

- 新建待办。
- 编辑待办标题和备注。
- 完成 / 取消完成待办。
- 删除待办。
- 设置截止时间。
- 设置优先级。
- 按完成状态筛选。
- 本地保存待办数据。

#### 建议支持

- 今日待办视图。
- 逾期待办提示。
- 按优先级排序。
- 待办与笔记关联。

#### 暂不支持

- 日历复杂视图。
- 系统级通知提醒。
- 周期性任务。
- 团队任务分配。

### 5.3 Markdown 功能

#### 必须支持

- Markdown 编辑。
- Markdown 预览。
- 编辑 / 预览切换。
- 标题。
- 有序列表和无序列表。
- 任务列表。
- 代码块。
- 行内代码。
- 链接。
- 加粗。
- 引用。

#### 建议支持

- 分屏编辑与预览。
- 代码块基础高亮。
- 预览滚动位置保持。

#### 暂不支持

- 所见即所得富文本。
- 复杂图表语法。
- 在线图片上传。

### 5.4 本地存储

#### 必须支持

- Windows 和 Android 均可离线使用。
- 笔记、待办、标签、主题配置均保存在本地。
- 应用重启后数据不丢失。
- 每条数据记录创建时间和更新时间。
- 删除数据使用软删除，便于同步时处理冲突。

#### 建议支持

- 本地数据导出为 JSON。
- 本地数据从 JSON 导入。
- 数据库版本迁移。

### 5.5 局域网同步

#### 必须支持

- 电脑和手机连接同一个 Wi-Fi 后可以同步数据。
- 支持手动发起同步。
- 支持通过 IP 地址和端口连接另一台设备。
- 同步笔记、待办、标签和主题方案。
- 支持基础冲突合并。

#### 推荐第一版交互

1. Windows 端点击“开启同步”。
2. 应用显示当前设备的局域网地址和端口。
3. Android 端输入地址。
4. Android 端点击“同步”。
5. 两端交换数据并合并。
6. 同步完成后展示结果，例如新增、更新、删除数量。

#### 冲突处理原则

- 同一条数据两端都存在时，以更新时间较晚的一方为准。
- 一端删除、一端更新时，比较删除时间和更新时间，较晚者生效。
- 不同设备创建的新数据直接合并。
- 同步失败时不覆盖本地数据，并提示用户重试。

#### 后续增强

- 二维码连接。
- 局域网自动发现。
- 同步历史记录。
- 同步前手动备份。

### 5.6 主题自定义

#### 必须支持

- 设置背景色。
- 设置主按钮色。
- 设置文字色。
- 保存主题方案。
- 切换主题方案。
- 应用重启后保留主题。

#### 建议支持

- 预设主题：极简白、深色、护眼绿、淡紫、灰蓝。
- 主题实时预览。
- 恢复默认主题。

#### 设计要求

- 自定义不能破坏可读性。
- 按钮、输入框、列表、Markdown 预览都应跟随主题变化。
- 当用户选择低对比度颜色时，应用应给出温和提醒。

### 5.7 简洁界面

#### Windows 端

- 推荐左侧为导航栏和列表区。
- 右侧为编辑区或详情区。
- 笔记编辑区应有标题输入、正文输入、预览切换。
- 待办区应支持快速添加和状态筛选。
- 设置区集中管理主题和同步。

#### Android 端

- 推荐底部导航切换笔记、待办、设置。
- 笔记列表和编辑页分屏改为页面跳转。
- 操作按钮保持少而明确。
- 避免复杂弹窗，优先使用底部面板或独立页面。

#### 交互原则

- 常用操作最多两步完成。
- 删除操作需要确认或支持撤销。
- 搜索入口明显。
- 空列表时展示可直接创建内容的入口。
- 同步、保存、删除等状态要有明确反馈。

## 6. 版本规划

### 6.1 MVP 版本

目标：完成一个可演示、可真实使用的本地笔记待办应用。

- Windows / Android 双端运行。
- 笔记新建、编辑、删除、搜索。
- 标签或分类。
- 待办新建、完成、删除、截止时间、优先级。
- Markdown 编辑和预览。
- SQLite 本地存储。
- 背景色、按钮色、文字色自定义。
- 手动局域网同步。
- README、截图和演示视频。

### 6.2 1.1 版本

目标：提升体验和稳定性。

- 二维码连接同步。
- 同步结果详情。
- 数据导入导出。
- 预设主题。
- 今日待办。
- 置顶笔记。
- 数据库迁移。

### 6.3 1.2 版本

目标：增强长期使用价值。

- 局域网自动发现。
- 待办与笔记关联。
- 同步历史。
- 本地备份。
- Markdown 代码高亮。
- 更完善的键盘快捷键。

## 7. 非功能需求

### 7.1 性能

- 常规列表操作应在 300ms 内响应。
- 1000 条笔记和 3000 条待办以内保持流畅。
- Markdown 预览不应明显卡顿。

### 7.2 可靠性

- 本地数据写入失败时应提示用户。
- 同步失败不应导致数据丢失。
- 删除数据应使用软删除。
- 应有基础异常处理和日志。

### 7.3 安全与隐私

- 默认不上传任何数据到云端。
- 同步只在用户主动操作后开启。
- 局域网同步服务关闭应用后停止。
- 后续可加入同步连接确认码，避免同网络陌生设备误连。

### 7.4 可维护性

- UI、业务逻辑、数据访问、同步逻辑分层。
- 数据模型统一定义。
- 同步协议使用 JSON，便于调试。
- 代码目录按功能模块组织。

## 8. 验收标准

### 8.1 笔记验收

- 可以创建一篇 Markdown 笔记。
- 可以编辑标题和正文。
- 可以预览 Markdown 渲染结果。
- 可以通过关键词搜索到笔记。
- 关闭并重新打开应用后笔记仍存在。

### 8.2 待办验收

- 可以创建待办。
- 可以标记完成。
- 可以设置截止时间和优先级。
- 可以删除待办。
- 关闭并重新打开应用后待办仍存在。

### 8.3 主题验收

- 可以修改背景色、主按钮色、文字色。
- 修改后主界面立即生效。
- 可以保存为主题方案。
- 重启应用后主题仍生效。

### 8.4 同步验收

- Windows 与 Android 在同一 Wi-Fi 下可以建立连接。
- 一端新增的笔记可以同步到另一端。
- 一端修改的待办可以同步到另一端。
- 一端删除的数据同步后另一端也显示为删除。
- 同步失败时本地已有数据不丢失。

## 9. 项目展示重点

用于实习投递时，可以重点展示：

- 独立完成 Windows / Android 双端应用。
- 使用 Flutter 实现跨端 UI 和主题系统。
- 使用 SQLite 实现本地优先的数据存储。
- 设计局域网同步流程和基于时间戳的数据合并策略。
- 支持 Markdown 编辑与预览。
- 完成从需求设计、产品交互、开发实现、测试验证到打包展示的完整流程。
